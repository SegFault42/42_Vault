Guidelines



Voici le barème du rush.

Il est important d'avoir à l'esprit que ce rush a pour but de vous faire exploiter les notions que vous avez déjà vues et vous les faire appliquer afin d'aboutir à un site fonctionnel.

Preliminaires


Classiquement, on git clone et evalue que ce qui se trouve sur la vogsphere.


Les verifications de base



Comme demande par le sujet :

- il doit y avoir un rendu

- il doit y avoir un fichier auteur

- les fichiers index.php et install.php doivent etre presents

- l'ensemble du rendu doit pas depasser les 5Mo.

- il n'y a rien d'interdit (POO, templates, etc..   voir le sujet).























L'essentiel


On teste les fonctionnalites attendues.


Ca tourne



Apres le git clone, demandez a l'étudiant corrigé de vous présenter les quelques manipulations qui permettent de mettre ce site en ligne.



Le site s'installe et se lance correctement. Vous y accedez normalement avec un browser (Chrome) sur le port du serveur web local.





















Vue principale



Le site doit ressembler au niveau structurel à un site de vente en ligne. Le client ne doit pas être perdu. La navigation sur le site doit être intuitive. Pour se faire, le client doit pouvoir voir du premier coup d'oeil :



- un visuel rapide du panier (nombre d'articles, somme totale)



- un accès aux informations du compte client ou un moyen de se connecter / s'inscrire



- la liste des catégories



Attention ! On ne juge pas la beauté mais plutôt l'ergonomie du site même si les deux restent intimement liés. 



















Gestionnaire de donnees



Le sujet propose deux façons de gérer les données dans ce projet. Soit par l'utilisation de mysqli, soit par un gestionnaire homemade.



Dans les deux cas, la notation se fera sur les même critères.

- On peut ajouter des données

- On peut lire des données

- On peut modifier des données

- On peut supprimer des données



















Gestion utilisateur



- On peut s'inscrire et se connecter (session)

- On peut modifier les informations clientes et supprimer ou désactiver le compte

- On peut remplir le panier sans être connecté puis au moment de la validation du panier, demande de connexion pour associer la commande au compte client



















Panier



- Le panier affiche bien la liste des produits sélectionnés avec la quantité, le prix et la somme totale.

On peut également valider la commande (archiver).



On peut annuler le panier et/ou modifier (articles, quantités) le panier avant validation



















Categories et Produits



- On peut voir plusieurs catégories

- Chaque catégorie possède un ou plusieurs produits

- Un produit possède un prix unitaire et peut appartenir à plusieurs catégories

- On peut ajouter des produits au panier et on peut modifier la quantité de chaque produit avant l'ajout



















Administration



- Il existe une zone d'administration sur le site où seul un ou plusieurs utilisateurs peuvent y accéder

- On peut gérer (création, suppression, édition) les utilisateurs

- On peut gérer (création, suppression, édition) les catégories et les articles

- On peut gérer les commandes des clients (création, suppression, édition)























Bonus




C'est cool les bonus !



Il y a vraiment de nombreuses possibilites. A vous d'evaluer leur importance selon le nombre, la complexite, etc...



- Gestion des stocks. Certains produits peuvent être indispo, etc...)

- Déclinaisons des articles, certains articles ont plusieurs déclinaisons (couleurs, ...)

- Informations utilisateurs supplémentaires (adresse livraison/facturation, ...)

- Captcha sur les formulaires à risque pour contrer les bots

- API / webservices

- Génération de graphiques/statistiques

- Génération de factures imprimables

- Utilisation de Oauth

- etc... (decrivez d'autres types de bonus que vous rencontrez dans le commentaire)
