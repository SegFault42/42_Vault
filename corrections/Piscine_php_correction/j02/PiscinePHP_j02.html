    Guidelines
    

    
Again :

    

    Realisez des tests simples rapides et efficaces.

    Normalement vous avez teste plein de trucs pour vos propres exos, donc re-testez les memes.

    

    ATTENTION: ce bareme ne va pas vous dire quels tests effectuer.

    Oui, certains feront peut-etre plus(+) de tests bizarres, et donc vautrer plus souvent les programmes (et arreter l'evaluation). Solution: multipliez les soutenances pour ecreter le phenomene.

    

    Des qu'un exo est faux, la notation s'arrete, l'evaluation s'arrete, on ne note pas les exos suivants. Vous pouvez toujours discuter entre vous des autres exos, mais en tout cas, on note rien dans l'intra. C'est egalement note dans les consignes du sujet, au point numero 7.

    Preliminaires
    

    

    De base
    

    
Classiquement:

    - soutenance en presentiel

    - git clone quand tout le monde est la

    - pas de rendu : 0

    - sauf si c'est precise, une erreur php temoigne d'une chose mal geree dans le code. Donc l'exo vaut 0 et l'evaluation s'arrete. (c'est comme un message d'erreur a la compil ou un segfault en C).

    

    

    

    

    





    

    

    

    

    Exo 00
    

    

    Autre monde
    

    
On teste avec un seul parametre, contenant plein d'espaces et de tabulations pour separer les mots.

    On controle que l'affichage nous renvoie que les mots, separes par un seul espace. Rien devant ni derriere.

    

    

    

    

    





    

    

    

    

    Exo 01
    

    

    Autre temps
    

    
Passer en parametre du programme :

    - soit une string qui ne suit pas le format decrit (jour_semaine num_jour mois an heure:minute:seconde)

      => le message d'erreur doit s'afficher

    - soit une string qui suit le format decrit dans le sujet, et qui represente un vrai moment temporel (donc pas d'heure 25, ni d'an fantaisiste avant 1970 ou apres 2038, ni de jour de la semaine qui colle pas avec la date, etc...)

    

    Controler avec le binaire date pour afficher la date a partir du nb de sec. calcule par php.

    Attention a la timezone avec le php comme avec date. Verifier au moins que ca colle avec l'exemple du sujet.

    

    On fait pas chier pour les accents. Si il y sont, c'est tres bien.

    

    

    

    

    





    

    

    

    

    Exo 02
    

    

    La loupe
    

    
Ca grossit ?

    

    Les 2 fichiers fournits en ressources doivent etre testes, a minima.

    Maintenant, passer la home d'un vrai site, c'est mieux. Essayez donc www.42.fr par ex.

    Et d'autres egalement (mefiez-vous, certain sites font de nombreux renvois par plusieurs pages avant d'afficher ce que vous voyez, donc un curl sur une home va pas vous donner toujours satisfaction, et aussi il y a parfois des liens dans des iframes, ou crees dynamiquement par du JS...  Tout ca, on s'en occupe pas. On reste sur les balises A dans le code html).

    

    

    

    

    

    





    

    

    

    

    Exo 03
    

    

    Who are you ?
    

    
controlez The Who.

    

    L'affichage doit etre identique a la commande who du systeme.

    Le script php ne doit pas utiliser d'une facon ou d'une autre le who du systeme (pas de exec_shell, de exec tout cours, de backquotes, ...) mais bel et bien ouvrir le fichier utmpx lire les enregistrements, les decoder et les afficher.

    

    

    

    

    

    





    

    

    

    

    Exo 04
    

    

    Livre photo
    

    
Aspirez moi donc tout ca !

    [j'ai pas dit sniffer, hein ?!]

    

    Vous avez du essayer avec plein de sites divers et varies, continuez pour cette soutenance.

    Verifiez la creation du repertoire, et le bon download des images.

    Attention, prenez pas des sites trop compliques, dans le sens ou la verification sera plus compliquee elle aussi.

    A nouveau, certains sites passent par des pages intermediaires, et la premiere page recue ne contient rien ou juste un renvois. Soyez vigilants.

    Uniquement ce qui est dans des balises IMG et pas dans le css par exemple. (si qqun vous fait en plus le css, on accepte bien sur).

    

    Il est de bon ton d'essayer differentes formes d'URL qui pourraient etre presentes dans les images, comme les externes, les directes, les relatives.

    (=> image avec lien sur un autre site, image avec un lien sur le meme site mais en absolu, donc avec l'url complete http://www.......jpg, ou lien relatif, sans le nom du site, seulement le path /img/pict.jpg )

    

    

    

    

    





    

    

    

    

    Exo 05
    

    

    Dans le D'(e)ni
    

    
Acces aux infos

    

    L'idee est controler que les variables(hash) correspondants aux noms des colonnes du csv sont bien creees, avec leur contenu correct.

    La clef pour chaque enregistrement est le contenu de la colonne dont le nom est passe en second parametre.

    

    Testez au moins les exemples du sujet

    Testez egalement avec un mini fichier csv que vous creez vous-meme (3 colonnes, 4 lignes, avec des noms de header imprevisibles)

    Testez des exemples un peu compliques, avec plusieurs commandes et des appels de fonctions php (explode, etc..).

    

    Si eval() emet une erreur (comme dans l'exemple), c'est pas un pb sur le code rendu, donc on laisse faire et on continue.

    

    

    

    

    





    

    

    

    

    Exo 06
    

    

    Le parchemin
    

    
C'est l'heure des sous titres

    [ou les sous-titres sont a l'heure]

    

    Testez le fichier fournit pour verifier que les sous-titres sont de nouveau bien ordonnes.

    => il faut que les timestamps et leur texte associe soient toujours ensemble

    => que les timestamps n'aient pas bouge

    => que la numerotation soit refaite pour etre ordonnee.

    

    Ca doit gerer le cas ou il y a plusieurs lignes de texte pour un seul element

    => telechargez un fichier srt de votre serie preferee, changez quelques lignes, passez le programme php et comparez a l'original.
